# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 仓库性质

这是一个**纯学习笔记仓库**，不是代码项目：没有构建/测试/lint 流程，没有依赖管理，所有内容都是中文 Markdown 笔记 + `resource/` 下的参考素材（图片等）。

主题：Palantir AIP（特别是 Ontology）→ FDE 角色 → 企业 AI 落地方法论，三段递进。

## 文件组织约定

笔记使用**数字前缀**编排阅读顺序，对应 README.md 中声明的学习路径阶段：

- `01-` / `02-` / `03-` — 主线笔记，按阶段递进，每个主题一个文件
- `99-notes-and-insights.md` — 横切的碎片洞察、追问、待解决问题；按日期分节追加
- `README.md` — 学习路径索引和进度看板（`- [x]` / `- [ ]`）
- `resource/` — 外部素材（截图、图表），文件名保持原样不要重命名

**新增主线主题时**：使用下一个数字前缀（如 `04-…`），同时更新 README.md 的路径图和进度清单。**追加洞察时**：写入 `99-notes-and-insights.md`，按日期建立 `## YYYY-MM-DD` 小节。

## 内容写作约定（从现有笔记归纳）

- **结构**：每篇主线笔记顶部有"来源 + 参考链接"块，紧跟"一句话定义"，再展开正文；中英对照官方原文用 `>` 引用块呈现
- **类比驱动**：用阿威已有的技术栈（Java/Python/OOP/知识图谱等）类比解释 Palantir 概念，不从零讲起
- **对比表达**：差异点用"传统方式 vs 有了 X 之后"的对照代码块/示例展示（见 `01-palantir-aip-ontology.md` 的"数据表 → 业务世界"段落）
- **追问留痕**：产生的新问题用 `- [ ]` 留在 `99-notes-and-insights.md`，不要在主线笔记里悬置
- **进度同步**：完成某阶段笔记后，同步勾选 README.md 的进度清单项

## 协作时需注意

- 这个仓库的"工作"= 写/改 Markdown。不要为它建议引入构建脚本、CI、linter 等。
- 用户（阿威）正在按个人节奏研究，**主线笔记的事实性内容必须有出处**（官方文档/原文链接），洞察可以主观但放进 `99-`。
- 提到 Palantir 概念时优先参考 `01-palantir-aip-ontology.md` 顶部链接的官方文档，而不是凭训练数据回答。
